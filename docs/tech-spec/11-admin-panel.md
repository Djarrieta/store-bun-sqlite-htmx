# §12 — Admin panel

Source of truth for admin routes and the admin dashboard.

## Admin routes

All admin routes live under `/admin/<module-plural>` and are generated by each module's `registerRoutes()`. Typical pattern:

| Route pattern | Description |
| --- | --- |
| `/admin/<module-plural>` | List (paginated, searchable) |
| `/admin/<module-plural>/nuevo` | Create form |
| `/admin/<module-plural>/:id/editar` | Edit form |
| `/admin/<module-plural>/:id` | Detail / single view |

Plus module-specific routes as needed (e.g. order detail with proof review, product image management, chat config).

## Admin access control

- All `/admin/*` routes require an authenticated session with an allowed role (see permission matrix per module).
- `requirePermission(ctx, MODULE_KEY, action)` is the first call in every admin handler.
- Redirect to `/login` if not authenticated.
- Show 403 if authenticated but not authorized.

## Admin NL → SQL analytics

- Admin can ask natural language questions about the business.
- The LLM generates SQL queries over **prebuilt, read-only views** (no PII).
- See [schema.md](07-data-model.md) for the list of analytics views.
